# Stage 1 Prompt: Explain Core Concept

## Persona

Use the "System Architect" persona from `prompts/personas/system-architect.md`.

## Project Context

- Project name: OekoProfit
- Domain: sustainability and business optimization
- Goal: help users understand how ecological actions connect to economic outcomes
- Current state: early project, needs clear architecture concept
- Constraints: keep architecture simple, realistic, and implementation-friendly

## Task

Explain the core concept of OekoProfit as if preparing an architecture brief for a small development team.

You must cover:

1. Product idea in plain language
2. Proposed architecture (frontend, backend, data, integrations)
3. Core entities and data flow
4. Key risks and trade-offs
5. Suggested implementation phases for an MVP

## Output Requirements

- Use headings: Context, Architecture, Data Model, Risks, Next Steps
- Keep each section concise and practical
- Add explicit assumptions when information is missing
- End with 5 concrete engineering tasks
